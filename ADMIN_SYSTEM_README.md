## 🎯 주요 기능

### 1. 배너광고 시스템

#### 위치 옵션
- **좌측 (Left)**: PC 화면 왼쪽에 세로로 3개씩 표시
- **우측 (Right)**: PC 화면 오른쪽에 세로로 3개씩 표시
- **상단 (Top)**: 페이지 상단에 가로로 표시
- **하단 (Bottom)**: 페이지 하단에 가로로 표시

#### 자동 회전 기능
- 30초마다 자동으로 위로 스크롤
- 좌/우측: 3개씩 보이고 나머지는 자동 회전
- 배너 번호(1, 2, 3, 4, 5...)로 표시

#### 지원 포맷
- JPG, PNG, GIF 이미지 파일

#### 설정 옵션
- 페이지별 표시 설정 (전체/메인/병원검색/간병인/재활운동/커뮤니티)
- 우선순위 설정
- 시작일/종료일 설정
- 클릭 링크 URL 설정
- 활성화/비활성화 토글

#### 모바일 최적화
- 모바일에서는 배너가 자동으로 숨겨짐 (좌/우 배너)
- 상/하 배너는 모바일에서도 표시 (크기 자동 조정)

### 2. 공지사항 시스템

#### 메인 메뉴 추가
- 모든 페이지 상단 네비게이션에 **"공지"** 메뉴 추가
- PC/모바일 모두 지원

#### 카테고리
- 서비스 안내
- 시스템 점검
- 이벤트/프로모션
- 법령·정책 변경
- 긴급(응급) 안내

#### 중요도 레벨
- **일반**: 회색 배지
- **중요**: 노란색 배지
- **긴급**: 빨간색 배지

#### 다국어 지원
- 🇰🇷 한국어
- 🇺🇸 English
- 🇨🇳 中文
- 🇯🇵 日本語

#### 콘텐츠 유형
- 텍스트 (기본)
- 이미지 첨부 가능
- YouTube 동영상 임베드 가능

#### 기능
- 상단 고정 (Pinned)
- 날짜 범위 설정
- 필터링 (전체/긴급/중요/카테고리별)
- 활성화/비활성화

#### 사용자 페이지
- **목록**: https://cashiq.org/notice/index.html
- **상세보기**: 자동 생성 (공지 클릭 시)

### 3. 팝업 시스템

#### 크기 조정
- 너비: 200~1200px
- 높이: 200~1000px
- 모바일에서 자동으로 화면 크기에 맞춰 조정

#### 콘텐츠 유형
- 텍스트
- 이미지
- YouTube 동영상

#### 표시 설정
- 페이지별 표시 (전체/메인/병원검색/간병인/재활운동/커뮤니티)
- 시작일/종료일 설정
- 활성화/비활성화

#### 사용자 경험
- "오늘 하루 보지 않기" 체크박스
- 닫기 버튼 (우측 상단)
- 배경 클릭으로 닫기
- 부드러운 애니메이션 효과

## 💻 사용 방법

### 배너광고 등록하기

1. **관리자 로그인** → **배너광고 관리** 클릭
2. 등록 폼 작성:
   - 배너 제목 입력
   - 위치 선택 (좌/우/상/하)
   - 이미지 URL 입력 (예: `https://example.com/banner.jpg`)
   - 링크 URL 입력 (선택사항)
   - 표시 페이지 선택
   - 우선순위 설정 (낮을수록 먼저 표시)
   - 시작일/종료일 선택 (선택사항)
   - 활성화 체크
3. **저장** 클릭
4. 메인 사이트에서 즉시 확인 가능

### 공지사항 등록하기

1. **관리자 로그인** → **공지 관리** 클릭
2. 등록 폼 작성:
   - 제목 입력
   - 카테고리 선택
   - 언어 선택
   - 중요도 선택
   - 한줄 요약 입력 (선택사항)
   - 내용 입력 (필수)
   - 이미지 URL (선택사항)
   - 동영상 URL (선택사항, YouTube 링크)
   - 시작일/종료일 (선택사항)
   - 상단 고정 체크 (필요 시)
   - 활성화 체크
3. **저장** 클릭
4. 공지 메뉴에서 즉시 확인 가능

### 팝업 등록하기

1. **관리자 로그인** → **팝업 관리** 클릭
2. 등록 폼 작성:
   - 팝업 제목 입력
   - 유형 선택 (텍스트/이미지/동영상)
   - 내용 입력
   - 이미지 URL 또는 동영상 URL 입력
   - 너비/높이 설정 (픽셀)
   - 시작일/종료일 설정 (필수)
   - 표시 페이지 선택
   - 활성화 체크
3. **저장** 클릭
4. **미리보기** 버튼으로 확인 가능
5. 해당 페이지 방문 시 자동으로 표시

## 🛠 기술 구현

### 데이터 저장
- 현재: `localStorage` 사용 (브라우저 기반)
- 관리자가 등록한 데이터는 브라우저에 저장되어 즉시 반영
- 프로덕션 환경에서는 서버 API로 교체 권장

### 파일 구조
```
/admin/
├── index.html        # 로그인 페이지
├── dashboard.html    # 대시보드
├── banners.html      # 배너 관리
├── notices.html      # 공지 관리
└── popups.html       # 팝업 관리

/notice/
├── index.html        # 공지 목록
└── view.html         # 공지 상세

/data/
├── banners.json      # 배너 데이터
├── notices.json      # 공지 데이터
├── popups.json       # 팝업 데이터
└── members.json      # 회원 데이터 (준비)

/js/
├── banner-display.js # 배너 표시 시스템
└── popup-display.js  # 팝업 표시 시스템
```

### 인증 시스템
- 세션 기반 인증 (`sessionStorage`)
- 관리자 로그인 후 세션 유지
- 로그아웃 시 세션 삭제
- 미인증 시 자동으로 로그인 페이지로 이동
